# Dulcitienda Convex Backend

> B2B Wholesale Distribution Platform Backend
> Colombia-focused confectionery, snacks, and beverages distributor

## Project Overview

Dulcitienda is a B2B wholesale e-commerce platform built for distributing confectionery products, snacks, beverages, and related goods to businesses in Colombia. The backend is powered by [Convex](https://convex.dev), providing real-time data synchronization, automatic scaling, and type-safe database operations.

### Key Features

- **Real-time Inventory Management**: Live stock tracking across warehouses
- **B2B Order Processing**: Wholesale order management with tier pricing
- **Product Catalog**: 5000+ SKUs across 10+ categories
- **Customer Management**: Company-based accounts with credit limits
- **Order Lifecycle**: Full order status tracking from pending to delivered
- **WhatsApp Integration**: Phone numbers for order notifications

## Quick Start Guide

### Prerequisites

- Node.js 18+ installed
- npm or yarn package manager
- Convex account (sign up at [convex.dev](https://convex.dev))

### Installation

1. **Clone the repository**:
```bash
git clone <repository-url>
cd dulcitienda-app
```

2. **Install dependencies**:
```bash
npm install
```

3. **Set up Convex**:
```bash
npx convex dev
```
This will:
- Authenticate with Convex
- Create a new Convex project (or link existing)
- Push schema to the cloud
- Start the dev server

4. **Seed the database** (optional):
```bash
# Seed with sample data (~60 products)
npx convex run seed:seedAll

# Or seed with massive catalog (500+ products)
npx convex run seedMassive:seedMassive
```

5. **Start development**:
```bash
# Start both frontend and backend
npm run dev

# Or separately:
npm run dev:frontend  # Next.js dev server
npm run dev:backend   # Convex dev server
```

### Environment Variables

Create a `.env.local` file in the project root:

```env
# Convex Configuration
NEXT_PUBLIC_CONVEX_URL=https://your-deployment.convex.cloud

# Optional: For production deployments
CONVEX_DEPLOY_KEY=your_deploy_key
```

| `npm run dev:frontend` | Start Next.js dev server only |

| `npm run start` | Start Next.js production server |
| `npm run lint` | Run ESLint (excludes _generated) |

### Convex CLI Commands

```bash
# Development
npx convex dev                    # Start dev mode (watches for changes)
npx convex dev --until-success   # Run until first successful push

# Dashboard
npx convex dashboard             # Open Convex dashboard in browser

# Data Operations
npx convex run <function>        # Run a mutation function
npx convex import <table> <file> # Import data from JSON/CSV
npx convex export <table>        # Export table data

# Deployment
npx convex deploy                # Deploy to production
npx convex deploy --preview      # Deploy to preview environment

# Codegen
npx convex codegen               # Regenerate _generated types
```

## Architecture

### Data Flow

```
Frontend (Next.js)
       ↕ (WebSocket - Live Queries)
Convex Backend
   ├── Query Functions (read-only)
   ├── Mutation Functions (write operations)
   └── Database (Document Store)
```

### Key Design Patterns

1. **Real-time Sync**: All queries are live by default
2. **Optimistic Updates**: Mutations update UI immediately
3. **Type Safety**: Full TypeScript coverage with generated types
4. **Pagination**: Cursor-based pagination for large lists
5. **Indexing**: Strategic indexes for query performance
